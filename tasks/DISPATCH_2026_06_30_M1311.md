# M1311: 渲染验证 — 31万行代码,屏幕上还是灰色方块

管理者诊断：build 成功,PBR shader 语法正确,drawArrays 调用存在。
但问题是**端到端数据流断裂** — composite_params.json 的数据没有正确流入 GPU passes。

核心问题清单：
1. verify-render.html 能画58个彩色方块 → 证明数据存在,WebGL2 能用
2. /world 页面黑屏或灰色 → GPURenderLoop 的 pass chain 在某处断了
3. drawCalls 显示 0 → 某个 pass 的 init 抛了异常被 catch 吞掉了

## 今日方法论：不写新代码,只做验证和修复

参考项目：
- google/swissgl — <1000行的WebGL2封装,能跑cellular automata
- ingewortel/artistoo — config → step() → draw() 模式
- cells2pixels.github.io — SwissGL 驱动的 NCA demo,每帧 1个step shader + 1个render shader

我们的问题不是缺少 shader 或 pass,是太多了。208个GPU模块,没有一个被验证过能独立工作。
